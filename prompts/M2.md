## Scope (PRD §11 M2)
Uncoached cash end-to-end: a player creates a table, players buy in from
bankroll, RNG hands play in a loop with full recording, players leave with
their stacks banked. The coach does not exist in this milestone except as
a regular player.

### 1. Graduate pure modules (legacy → server/)
bettingRound, Deck, HandEvaluator, ShowdownResolver, SidePotCalculator,
positions. Per the PRD §9 bar: each gets a NEW test suite written from
poker rules and the specs (never from old behavior), zero old-world
imports, then MOVE out of legacy/. HandGenerator/RangeParser stay in
legacy (M4). ReplayEngine stays (M6).

### 2. Lean game core (the GameManager successor — rebuild, never port)
- One state object per table; pure modules injected; phases:
  waiting → preflop → flop → turn → river → showdown → hand_complete.
- Betting flow via bettingRound; side pots via SidePotCalculator;
  showdown via ShowdownResolver/HandEvaluator; button/blind rotation via
  positions. Heads-up blind rules correct.
- **Design for M4 without building it:** the deal step is a seam — M2
  wires only an RNG dealer, but the interface must accept a future card
  source (DEALING.md §3 awaiting_deal will plug in here). Do NOT
  implement manual dealing.
- RUNTIME §1 discipline: persist stacks after every completed hand; on
  boot, rebuild tables from snapshots and VOID any in-flight hand (bets
  returned to hand-start stacks, nothing recorded).

### 3. Uncoached mode behavior (RUNTIME §2–4)
- Table creation from lobby by any player: SB/BB, table size (6/9),
  optional name. Buy-in on join: 50–250 BB (default 100), debited from
  bankroll via the M1 atomic function (type buy_in); insufficient balance
  blocks with a clear error.
- Auto-start at 2 seated players; ~2s pause between hands; sit-out /
  sit-in; bust → offered re-entry (new buy-in) or seat released.
- Action timer 30s, auto check/fold. Disconnect: 60s grace → auto
  sit-out; 5 min retention → auto cash-out (type cash_out) and seat
  release. Leave table / table close → cash out remaining stack.
- Idle table (zero connected players 10 min) → close, cash everyone out.

### 4. Recording (CONTRACT §4.3–4.4 shapes, storage-side)
- Migrations: sessions, hands, hand_participants, hand_actions.
- A session opens at table activation and finalizes at table close
  (completed-only principle). Store `crm_entry_id` null for uncoached.
- Every completed hand: board, pot, full action list (seq, street,
  action, amount), per-participant stack_start/stack_end, position,
  hole_cards, is_winner, and the booleans/opportunity counters:
  vpip, pfr, three_bet_opp, three_bet, saw_flop, cbet_opp, cbet, wtsd,
  wsd. Definitions matter — write them as documented code comments and
  test them against scripted hands.
- origin = 'rng' always in M2 (hardcode the value, keep the column enum
  full per CONTRACT).
- Voided hands are never written. No tags (M5), no export endpoints (M3).

### 5. Client
- Lobby: create-table flow + join with buy-in dialog.
- Table page: seats, board, pot, betting controls, timer display,
  sit-out toggle, leave. Audit legacy/client/PokerTable.jsx +
  BettingControls.jsx per PRD §9 — graduate if they pass the bar,
  rebuild if not; either way log the verdict in docs/decisions/.
- Reconnect: rejoining the app while seated returns you to your table
  with live state.

## Acceptance criteria
1. E2E test: 3 players create/join/buy in, play ≥10 scripted hands
   (deterministic seeded deck for tests), sit-out, one busts and
   re-enters, all leave — bankroll ledger reconciles exactly: for every
   player, sum(buy_ins) - sum(cash_outs) == net stack change, and table
   chips always equal sum of buy-ins minus cash-outs.
2. Counter correctness: scripted hands with known VPIP/PFR/3bet-opp/
   cbet-opp/WTSD outcomes assert every boolean.
3. Crash test: kill the server mid-hand, boot, the hand is voided, bets
   refunded, no hand row written, next hand plays normally.
4. Disconnect test: socket drop → sit-out at 60s → cash-out at 5 min,
   ledger correct.
5. Zero remaining imports from legacy/ for graduated modules; legacy/
   contains only M4+ material. npm test green.
